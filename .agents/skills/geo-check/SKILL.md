# GEO Check

用于对网页与站点做 GEO（Generative Engine Optimization）检查，并给出整改优先级。

## 适用场景

- 用户要求检查网站是否更容易被 AI 引用（如 Google AI Overview / Copilot / Chat 结果）。
- 用户希望把 SEO 工作升级为 GEO 工作流。
- 用户需要一份可执行的上线前检查单（内容、技术、可抓取性、可引用性）。

## 核心原则

1. 好 SEO = 好 GEO（以可抓取、可索引、可理解为基础）。
2. 不为 AI 过度做“切块写作”或关键词堆砌；先保证真实价值与可验证信息。
3. 强化“可被引用”的页面特征：明确事实、来源、结构化信息、稳定 URL。
4. 以“被引用质量 + 转化质量”作为关键衡量，而不是只看流量。

## GEO 检查点（执行顺序）

### P0：可抓取与索引基础（必须先过）

- robots/sitemap 正常可访问，返回 200。
- 重要页面可索引（无误设 noindex / 错误 canonical）。
- 站点核心页面具备稳定 URL，不频繁变更路径。
- hreflang 与 canonical 自洽（双语或多语场景）。
- 首屏与关键内容可被渲染后抓取（避免关键信息仅在交互后出现）。

### P1：内容可引用性（GEO 关键）

- 页面回答明确：主题、结论、约束、适用条件清楚。
- 非同质化内容：包含经验性信息、对比、边界条件、失败案例或数据依据。
- 段落结构清晰，便于模型抽取引用（定义、步骤、注意事项分层）。
- 关键结论配来源或证据（原始文档、官方链接、数据时间）。
- 避免空泛营销语；每页有“可复述事实”。

### P1：实体与结构化信号

- 标题、描述、H1 与正文语义一致，不误导。
- 关键页面补充结构化数据（按页面类型选择组织、FAQ、HowTo、SoftwareApplication 等）。
- 品牌实体信息一致（名称、官网、社媒、联系方式等）。
- 图片/媒体有可读 alt 与上下文，不做纯装饰堆叠。

### P2：AI 检索协同能力

- 提供 AI 友好文本入口（如 `llm.txt` 或等价“全量说明页”），并与站内内容一致。
- 产品/工具页参数说明清晰，便于 query fan-out 场景下被正确匹配。
- 关键任务页存在“对比/选择建议”，帮助模型在推荐环节引用你。

### P2：监测与反馈闭环

- 在 Bing Webmaster 接入并观察 AI 报告：引用份额、主题覆盖、查询意图。
- 按页面簇追踪“被引用页面 -> 到站行为 -> 转化行为”。
- 区分品牌词与非品牌词的 GEO 表现，优先补齐非品牌高意图问题页。

## 输出模板（交付给用户）

1. 结论摘要（当前 GEO 就绪度：高/中/低）
2. 风险清单（P0/P1/P2）
3. 修复任务表（任务、负责人、预计工作量、验证方式）
4. 指标看板（引用份额、引导访问质量、转化率、页面覆盖率）

## 执行注意

- 不把第三方“GEO 分数”当作唯一依据。
- 不建议为迎合 AI 输出格式而牺牲真实可读性。
- 所有优化优先以用户问题解决效率与内容真实性为准。
